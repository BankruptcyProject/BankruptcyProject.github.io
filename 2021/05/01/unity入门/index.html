网格

每一个代表为1Unit（对应屏幕上的100像素点）

摄像机的几个参数

Z坐标：默认是-10，不用修改 Background，最底层背景色 Size广角，指摄影范围，默认5个单位 其中，5Unit为半高，全高为10Unit ,是摄影范围的高度 长宽比 在Game（游戏）窗口中调整 若要1080p分辨率：长1920像素点，宽1080像素点，则Size=5.4，比例为FH

游戏对象

隐藏/显示 操作在Inspector（检察器）上方的打勾栏 图层顺序越小在越底下 对象轴心（Pivot）修改：在下方文件文件夹中选择素材，右栏中打开sprite editor 父子关系：在左栏拖动完成。子坐标是相对于父的。对于父子AB组成的一个对象C，C的坐标轴心和旋转可为AB中心或A，在攻击栏切换轴心 一个游戏对象可以拥有不同素材/造型，在右栏sprite renderers可添加造型 sprite renderers是一个组件，功能是显示图片

图片素材

背景一般为jpg图片，其它一般为png图片。 素材切割：在下方文件文件夹中选择素材，在右栏sprite mode设为multiple，打开sprite editor，执行切割，切割后的素材展开可分别拖出。

游戏脚本

最好在下栏新建文件夹用于存放脚本 这是一个文件名叫Hello的脚本

将脚本挂载到游戏对象：于游戏对象的检查器中添加脚本组件，或者直接拖动 关于脚本： 1.类名必须和文件名相同 2.必须继承于MonoBehaviour，所有unity脚本都继承于这个类 3.默认定义了一些事件函数，如：Start(),Updata,会被unity引擎自动调用 脚本在引擎内部执行顺序： 1.创建游戏对象 2.创建组件，比如Hello sjtql = new hello(); 3.sjtql.Start() 4.定时执行sjtql.Updata()更新游戏显示 5.一个脚本可以挂载在多个对象上，一个对象也可以挂载多个脚本

预制体

1.先在场景上做好一个游戏对象 2.在层级管理器把游戏对象直接拖到Project窗口，则生成一个预制体资源*.prefab 3.给预制体改个名字，以免混淆，重要！！！！！

主预制体单独编辑，同步更新所有预制体；预制体的一个实例修改，可选应用到主预制体；可选择某个预制体的实例，使其脱离预制体成为一个单独的游戏对象

动态创建实例

1 添加脚本，挂载

2 添加公共变量 public GameObject myPrefab;

3 在 Inspector 中引用资源 预制体可添加到节点上

4 当点击鼠标时，创建一个子弹的实例

GameObject bullet = Instantiate(myPrefab);

bullet.transform.position = transform.position + new Vector3(0, 1f, 0);

其中，Instantiate() 方法根据一个 Prefab ，来创建一个实例，返回值就是创建出来的GameObject 。

5 运行游戏 在 Hierarchy 窗口中，观察新创建的实例

Instantiate() 重载了很多版本，常用的如下：

bullet = Instantiate(myPrefab)

创建一个实例，position与rotation与当前游戏对象相当，挂载场景根节点下

bullet = Instantiate(myPrefab, Transform parent )

创建一个实例，position与rotation与当前游戏对象相当，同时指定父节点

bullet = Instantiate(myPrefab, position, rotation)

创建一个实例，挂载场景根节点下，同时指定 position与rotation

例如，

bullet = Instantiate(myPrefab, transform.position, transform.rotation)

bullet = Instantiate(myPrefab, position, rotation, Transform parent)

创建一个实例，挂载 parent 节点下，同时指定 position与rotation
