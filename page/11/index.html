脚本中的MyComponent类是继承自MonoBehaviour类的，只要继承自MonoBehaviour类，那么该类就是一个组件类。
如果删掉父类或修改父类为其他类，那么这个脚本就是一个普通的C#脚本而非组件脚本。

脚本的生命周期

方法名称 调用时间
Awake 最早调用，所以一般可以在此实现单例模式
OnEnable 组件激活后调用，在Awake后会调用一次
Start 在Update之前调用一次，在OnEnable之后调用，可以在此设置一些初始值
FixedUpdate 固定频率调用方法，每次调用与上次调用的时间间隔相同
Update 帧率调用方法，每帧调用一次，每次调用与上次调用的时间间隔不相同
LateUpdate 在Update每调用完一次后，紧跟着调用一次
OnDisable OnEnable相反，组件未激活时调用
OnDestroy 被销毁后调用一次

unity多线程与协程IEnumerator

Posted on 2022-02-28 Edited on 2022-11-14

多线程

创建一个子线程。在unity中国子线程许多操作都是被禁止的，但仍可用于数据处理、网络信息的传输，读写文件操作和复杂的逻辑计算等。

比起多线程，unity更推荐使用协程来进行一些异步操作，但协程本质上是单线程。主线程停下会进入协程执行代码，知道协程宣布挂起，然后后主线程会回到主代码执行，协程则在挂起状态等待，知道等待时间结束。

协程注意事项

协程在当前主程序运行时,开启另一段逻辑处理,协程主要用于代码等待

协程的方法名最前面最好加上大写的IE表示这是一个协程

如果协程中有嵌套协程的时候,关闭当前协程不会影响嵌套的协程,只会关闭当前协程,嵌套协程继续执行

协程所在脚本关闭的时候协程不会停止,但是协程所在的游戏对象删除的时候协程将会被系统回收

协程的返回值必须是IEnumerator

协程的内部必须体验Yield return 值

协程可以重载

协程可以多次调用,但是不可以关闭后继续运行,只能重新开启协程

协程可以嵌套

协程的参数不能使用ref 和out 修饰

协程在执行完内部所有代码的时候会退出协程

关于yield return

yiled return 0;
0可以换成任意的int类型的值,都是当前暂停一帧,从下一帧开始执行

yiled return 协程;
yiled return 后面可以跟一个协程,当执行到该行代码的时候执行yiled return 后面的协程,如果下面还有代码在执行完该协程的时候,回来继续执行。

yiled return new WaitForSeconds(时间);
时间可以随意设置,该方法主要作用是等待几秒钟后执行,yiled return 后面的代码

yiled return 后面可以跟一个类,等等根据自己的需要进行使用

例

开启和关闭协程

1.直接使用协程方法名
开启协程 StartCoroutine(“协程方法名 “);
关闭协程 StopCoroutine(“协程方法名 “);

2.使用协程方法名加参数
开启协程 StartCoroutine(“协程方法名”,参数);
关闭协程 StopCoroutine(“协程方法名”);
//该方法只适用于协程只有一个参数的时候

3.使用方法
开启协程 StartCoroutine(协程方法名(参数));
关闭协程 StopCoroutine(协程方法名(参数));
//该方法中的参数个数随意
//该方法不能关闭该协程,但是开启可以使用
使用方法参数正确开启关闭协程:
<1>定义一个IEnumerator 类型的全局变量StopIE
public IEnumerator StopIE;
<2>接收 协程方法名(参数)
StopIE = 协程方法名(参数);
<3>开启协程
在这里开启协程可以直接用变量代替方法名加参数
StartCoroutine(StopIE);
<4>关闭协程
在这里同样
StopCoroutine(StopIE);

注： 1.动画分为旧版的动画（Animation）和新版的动画器（Animator）
2.unity一般使用FBX的模型和动画，如果希望进行复杂的建模操作，一般使用3DS MAX、Maya等进行动画制作，再导出为FBX格式。

动画组件（Animation）

重要参数:
动画：默认使用的动画剪辑，需要拖入

Play Automatically：勾选后会自动播放默认的动画剪辑
Animate Physics：勾选后动画将使用物理循环来驱动
Culling Type（剔除方式）
总是动画化：离开屏幕后动画也会执行

动画剪辑

动画器（Animator）

Animator用于管理大量的Animation以及Animation之间的切换和过渡，同时区分了普通动画和人物骨骼动画。

重要参数
控制器：该组件使用的动画控制器
Avatar（替身）：该动画使用的骨骼映射
应用根运动：是否使用动画本身的位移旋转
更新模式：设定动画何时更新

动画器面板中有初始三个节点：Entry代表入口节点，Any State代表任意状态节点，Exit代表退出节点。

动画器面板有参数：Float、Int、Bool、Trigger（单次触发类型，在代码中每触发一次，状态就过渡一次）。在节点间的过渡线中可以打开条件（condition）中，将参数作为状态过渡条件。

对于导入的动画来说 动画类型：包括：无（不导入动画）、旧版（旧版动画系统的动画）、泛型（通过动画，适合各种类型的动画）、人形
Avatar定义：骨骼动画的来源
Skin Weights（蒙皮权重）：设置骨骼数量和权重
剪辑：可在本动画基础上创建剪辑
循环时间：动画是否需要循环播放
循环动作：自动过渡首位动画使得动画更加流畅，出现绿色图标代表首尾动画非常匹配
根变换旋转：控制旋转是否烘培到骨骼动作中
根变换位置：控制移动是否烘培到骨骼动作中 （如走路旋转是否真的会在世界中产生位移）
可创建（播放与参数）的曲线，配合脚本使用
