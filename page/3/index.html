案例-Auction

Posted on 2022-09-27

NetworkServer.Spwan
这个函数只能在服务器调用。
NetworkServer.Spwan(Instatelate(GO))
其中这个函数重载在服务端生成一个GO，并在所有客户端各生成一个GO克隆。这些GO都没有Client Authority。
NetworkServer.Spwan(Instatelate(GO), NetworkConnection)
在某个客户端（也可以是主机）生成一个GO，并在所有客户端各生成一个GO克隆。在目标主机的GO有Client Authority。其他GO克隆没有Client Authority。

只有拥有Client Authority的对象能调用Cmd函数，Cmd函数会在这个对象的服务器克隆上执行。通过网络生成的克隆统统无Client Authority故无法调用Cmd函数。
Cmd函数中除了传递过来的参数，其它所有变量都是服务器上的 对象克隆 身上的。

Client Authority可以理解为真身。一个物体A通过网络生成，每个客户端上都会有一个A，其中只有一个是真的，其它都是克隆体。真身有Client Authority，克隆体无Clietn Authority。
在服务器上我们可以移除真身或修改哪个是真身，AssignClientAuthority，但真身最多只有一个。

假设物体A有Client Authority和Cmd函数，那么其它任何物体（不论有无联网）都可以在各自脚本里通过Find物体A来调用Cmd函数。因为本质上是A执行Cmd函数，不是其它物体执行该函数。

Network Start Position

Posted on 2022-09-08 Edited on 2022-09-09

要控制玩家的生成位置，可以使用“Network Start Position”组件。

若要使用这些，请将“Network Start Position”组件附加到场景中的游戏对象，并将游戏对象定位到希望其中一个玩家开始的位置。可以向场景添加任意多个开始位置。Network Manager检测场景中的所有开始位置，当生成每个玩家实例时，它使用其中一个的位置和方向。

默认情况下，Network Manager将在（0，0，0）生成玩家。将此组件添加到游戏对象将自动向Network Manager注册/取消注册其游戏对象的变换，作为可用的生成位置。

根据Network Manager的玩家spwan方法设置，spwan是随机的（可能两个或多个玩家将使用相同的spwan位置）或循环的（使用每个可用位置，直到客户端多于spwan点）。

Network Attributes

Posted on 2022-09-08 Edited on 2022-09-13

标记变量和方法

[Server]
只有服务器可以调用该方法（在客户端上调用时抛出警告）。

[ServerCallback]
Same as Server but does not throw a warning when called on client.

[Client]
只有客户端可以调用该方法（在服务器上调用时抛出警告）。

[ClientCallback]
Same as Client but does not throw a warning when called on server.

[Command]
从客户端调用此函数以在服务器上运行此函数。用于反作弊。不可能从服务器调用。
核心逻辑就是，当客户端的脚本调用此特性修饰的函数时，服务端的对应的对象执行此函数。任何允许数据类型的参数都将通过命令自动传递到服务器。
小心每帧从客户端发送命令！这可能会导致大量网络流量。

[ClientRpc]
服务器使用远程过程调用（RPC）在客户端上运行该函数。
核心逻辑就是，当服务器的脚本调用此特性修饰的函数时，客户端的对应的对象执行此函数。
这意味着函数中的代码不会在服务器执行。

[TargetRpc]
允许从服务器在客户端上调用它们。与ClientRpc属性不同，这些函数是在单个目标客户机上调用的，而不是在所有就绪客户机上。

可以获取到挂载在目标对象上的NetworkIdentity，可以通过此获取到目标客户端与服务器的连接的通道NetworkIdentity.connectionToClient。
将其作为参数TargetRpc的第一个参数由服务器执行，即可只发送给目标客户端。

[SyncVar]
SyncVars用于自动将变量从服务器同步到所有客户端。不要从客户端修改这些变量，这是没有意义的。不要让它们为空，否则会出错。当客户端从服务器接收更新时，可以使用SyncVar Hooks在客户端上运行代码。

NetworkBehaviour脚本用于具有Network Identity组件的游戏对象。这些脚本可以执行高级API功能，如命令、客户端RPC和同步变量。
这个组件NetworkBehaviour一般用于被其他脚本继承。

属性

isServer

isClient

isLocalPlayer

hasAuthority 如果此客户端控制此游戏对象，则在客户端上返回true。它在服务器上下文中没有意义。 netId 此游戏对象的唯一网络ID。服务器在运行时分配此值。它对于网络上的所有游戏对象都是唯一的。 netIdentity 返回此对象的网络标识 connectionToServer 连接上服务器，只对玩家有效，对克隆玩家无效 connectionToClient

Network Callbacks

Network Behaviour scripts 中内置 callback functions 它们是基类上的虚函数 可以在自己的代码上 override 它们。

方法 callback
OnStartServer called on server when a game object spawns on the server, or when the server is started for game objects in the Scene
OnStopServer called on server when a game object is destroyed on the server, or when the server is stopped for game objects in the Scene
OnStartClient called on clients when the game object spawns on the client, or when the client connects to a server for game objects in the Scene
OnStopClient called on clients when the server destroys the game object
OnStartLocalPlayer called on clients after OnStartClient for the player game object on the local client
OnStopLocalPlayer called on clients before OnStopClient for the player game object on the local client
OnStartAuthority called on owner client when assigned authority by the server. hasAuthority will be true for such objects in client context.
OnStopAuthority called on owner client when authority is removed by the server.

Network Transform

Posted on 2022-09-08

The Network Transform component synchronizes the position, rotation, and scale of networked game objects across the network.

该组件需要Network Identity组件

默认情况下Network Transform有服务器权限，除非专门勾选客户端选项。
客户端权限适用于玩家对象以及专门分配给客户端的非玩家对象，但仅适用于该组件。启用此功能后，位置更改将从客户端发送到服务器。

压缩勾选框Compression压缩传送数据，压缩四元数适合3d不适合2d

Authority：操作权限。如果Client Authority被勾上，那么只有在客户端改变物体Transform时才会同步Transform，且服务端无权改变物体的Transform；反之亦然。从反作弊的角度来看，通常不勾上是更好的选择，即只有服务端有权限改变物体的Transform

Sync Position / Sync Rotation：是否同步位置/旋转。如果不勾上的话，如果服务端改变的物体的位置/旋转，客户端将不会有反映。

Interpolation：是否进行补间动画。这个意思是，通常由于网络延迟等问题客户端可能出现丢包的情况，即没有来得及/能够同步服务端的位置/旋转。如果不勾上的话，物体会在客户端这里发射”瞬移“的情况。如果勾上的话，物体将会平滑地移动过去。

Network Manager

Posted on 2022-09-08 Edited on 2022-09-13

用于管理（游戏状态、角色生成位置、场景）。

多人联机的核心组件，创建一个空对象挂上该组件。

同时Network Manager组件不能于Network Identity组件共存于一个对象上。因为场景在进行本地加载时会禁用带有Network Identity组件的对象。

有Client模式、Server模式、Host模式。使用HUD可以直接选择，如果是自写UI需要在代码中选择模式
NetworkManager.StartClient
NetworkManager.StartServer
NetworkManager.StartHost
在客户端模式下，游戏会试图连杀NetworkManager中设置的ip和端口。在主机或服务器模式下，游戏会监听NetworkManager中设置的ip和端口。

Spwan

拖入本地控制的玩家预制体，该预制体需要有Network Identity。离线是将自动摧毁该对象。

除了玩家预制体外，还需将有其他有网络标识的对象注册进列表。
也可以通过代码注册，用ClientScene.RegisterPrefab方法.

一个场景有且只能有一个Network Manager。Network Manager有DDOL选项，勾选以贯穿所有场景。同时Network Manager身上注册的所有生成预制体也会跟着到下一个场景生成。如果勾选没有DDOL选项，则需要在每个场景上挂一个NetworkManager，并且每一个NetworkManager上要重新注册生成预制体。

生成位置 Choose Random to spawn players at randomly chosen startPosition options.
Choose Round Robin to cycle through startPosition options in a set list.
Network Start Postion组件可以自定义生成位置相关设置

Scene Management

当服务器或主机启动，Online场景会在所有客户端上加载。networkSceneName属性中存有被加载场景的名字。
断线时offine场景会被加载。
在代码中 ServerChangeScene方法可以改变场景，此时所有客户端都会跟着一起切换场景。ServerChangeScene的值会被更新
