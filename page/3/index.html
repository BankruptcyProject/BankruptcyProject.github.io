Lambda

Lambda表达式本质上就是匿名函数

使用lambda声明运算符 =>（读作goes to）的表达式就是Lambda表达式

创建Lambda表达式，需要在 Lambda 运算符左侧指定输入参数（如果有），然后在另一侧输入表达式或语句块

即是说我们使用形如 （）=> { } 这样的格式就可以创建Lambda表达式

任何 Lambda 表达式都可以转换为委托类型。
Lambda 表达式可以转换的委托类型由其参数和返回值的类型定义。
如果 Lambda 表达式不返回值，则可以将其转换为 Action 委托类型之一；否则，可将其转换为 Func 委托类型之一

如果是没有参数，必须有这个空括号
Action line = () => Console.WriteLine();

如果只有一个输入参数，则括号可有可无
Func<double, double> cube = x => x * x * x;

两个或更多输入参数使用逗号加以分隔
Func<int, int, bool> testForEquality = (x, y) => x == y;

有时，编译器无法推断输入参数的类型。 可以显式指定类型，如下面的示例所示：
Func<int, string, bool> isTooLong = (int x, string s) => s.Length > x;

泛型

泛型（Generic）允许编写一个可以与任何数据类型一起工作的类或方法。

where关键字

在声明泛型方法/泛型类的时候，可以给泛型加上一定的约束来满足我们特定的一些条件。

比如：

泛型限定条件：

  • T：结构（类型参数必须是值类型。可以指定除 Nullable 以外的任何值类型）
  • T：类 （类型参数必须是引用类型，包括任何类、接口、委托或数组类型）
  • T：new() （类型参数必须具有无参数的公共构造函数。当与其他约束一起使用时new() 约束必须最后指定）
  • T：<基类名> 类型参数必须是指定的基类或派生自指定的基类
  • T：<接口名称> 类型参数必须是指定的接口或实现指定的接口。可以指定多个接口约束。约束接口也可以是泛型的。

Network Identity

组件

加载场景顺序：
加载没有NetworkIdentity组件的结点
等待联网
当一个玩家联网时，加载（更新）所有最新状态的拥有NetworkIdentity组件的结点

运行逻辑与顺序

在本地客户端运行游戏，场景内会有多个人物，其中有一个人物由本地控制——称为玩家
另外几个人物也是本地生成，组件、材质、脚本都于本地运行，但它们的控制命令来自于服务器——称为克隆玩家
控制玩家实际上是向服务器发送行动请求，服务器许可后玩家及其对应的所有克隆玩家行动

Network Attributes

同步变量标记，被标记的变量会同步给其他玩家
那么一旦此变量(在服务器！！！！)发生变化，服务器就会将此变量发送给所有客户端
后面是同步变量被修改时的监听方法

Command标记是针对方法的标记，被标记的方法必须以Cmd开头
CmdSetupPlayer这个方法是被 [Command]标记过的，所以是会在服务器端执行的，不会再本地执行
如果取消了[Command] 的标志那就肯定只在本地执行了也就是完全的本地的方法啊和服务端没关系
在服务器中改变同步变量才会被分发给客户端！！！！！！！！！！！！！
Command方法描述为 会被客户端调用的服务端的方法

Bypassing Authority
为了安全起见, [Command] 只能在player object上使用
non-player objects想要使用[Command]需要拥有authority，或者使用[Command(requiresAuthority = false)]可以绕过权限检查（但无法传递参数）

关于标记更多信息参考以前写的博客Mirror_Network Attributes
