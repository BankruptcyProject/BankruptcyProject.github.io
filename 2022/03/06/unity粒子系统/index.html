unity粒子系统

Posted on 2022-03-06 Edited on 2024-04-02

Particle System 组件

包含1个主模块和22个子模块(可隐藏)

粒子系统由粒子和发射器组成，发射器创建并发射粒子

为了同时预览多个粒子组合效果，需要将多个粒子挂载在同一个父粒子身上，此时父粒子一般不启用发射模块

主模块

起始速度、起始大小、起始颜色等参数 可以在右边下拉栏中 切换成随机大小并设置随机区间或曲线

预热参数代表 在摄像头观察前先工作一周期

启动延迟（和循环、预热不能同时用）

发射器速度（发射器移动方式rig或trans）

是否随机（每次播放效果不同）

停止行动（播放完后是否会销毁自身对象）

剔除模式（粒子不在视野内的行动模式）

模拟空间 Simulation Space：Local以粒子发射器为中心，World使用世界坐标系。模拟空间控制着每个粒子的运动轨迹，因此设置为local后，粒子发射器移动将会影响已发射粒子，设置为World则仅影响未发射粒子

Renderer（渲染器）子模块

billboard模式，控制粒子始终面向摄像机，且粒子贴图初始头朝上

strectched billboard模式，粒子贴图初始朝向为 头(左)朝飞行方向，Length可调整粒子长宽比，CameraScale与SpeedScale考虑相机和粒子的速度进行拉伸，模拟粒子高速运动的效果

渲染器渲染粒子，可以设置材质，材质可以设置贴图

粒子着色器
材质 着色模式 可以改为 粒子着色模式

Emission（发射器） 子模块

匀速发射模式（默认）：
随单位时间产生的粒子
随发射器移动距离产生的粒子

突发（爆发）发射模式：
短时间内一次性发射出大量粒子

Shape（发射形状）子模块

关闭该模块，则发射器变成一个点，沿着一条线发射

Color Over Lifetime （生命周期内颜色）子模块

渐变色编辑器：
上面控制点控制Alpha，下面控制点控制Color
可增加、删除、移动控制点，最多可以有8个控制点

常用效果：淡入淡出

注：在主模块中也可以设置颜色，粒子最终颜色=初始颜色*渐变颜色

Size Over Lifetime （生命周期内大小）子模块

曲线编辑器，使用的是贝塞尔曲线

Velocity over lifetime 子模块

从x、y、z三个轴分别控制速度，可设置速度曲线

绕行效果：

设置轨道速度，此时xyz分别代表轨道轴，射线参数（Radial）代表离心速度

粒子速度受到主模块StartSpeed 和 子模块的同时影响（向量相加）

Force over lifetime 子模块

注意，如果要模拟重力效果，则需要使用世界坐标系

物理系统中的力 与 粒子系统中的力 相互独立互不影响

Limit velocity over lifetime 子模块

Drag：摩擦系数，模拟介质阻力

Multipy by Size：粒子越大、阻力越大

Multipy by Velocity：粒子越快、阻力越大

Noise 子模块

对粒子的随机扰动

设置扰动强度和扰动频率

案例

火星飞溅效果

Shape: Sphere球形

Emission: Burst爆发式

Main: Start Size 大 / Lifetime 短 / Size 小

Velocity: Speed Modifier 模拟衰减

Renderer: Stretched Billboard, Speed Scale 速度方向上拉伸

Color over lifetime: 淡入淡出，颜色白-火红-暗

粒子着色器

Renderer（渲染器）子模块 中选择粒子的材质，在材质中选择 Particle/standard Unlit 的 Shader

Rendering Mode（渲染模式）的 Additive（叠加）：粒子颜色将与背景颜色混合（(A+B)/2），当粒子贴图有黑色时候会使得黑色部分消失。

Color Mode 的 Multiply：粒子颜色=粒子贴图颜色*Albedo Color
一个颜色与黑色相乘，结果为黑色；一个颜色与白色相乘，结果为原颜色

应用：使用黑底白色的粒子贴图用于描述粒子的形状，启用Rendering Mode的Additive和Color Mode 的 Multiply，之后就可以在Albedo Color中自由描述粒子的颜色，同时配合color over lifetime更方便更直观的控制粒子颜色。

Trails（拖尾）子模块

拖尾是跟随在粒子后面的一个小纸片，起点为发射点，终点为粒子位置。

添加Trails子模块后，Renderer子模块中可以为拖尾添加材质

拖尾长度由lifetime决定（0~0.1），代表占 粒子生命周期Start lifetime 的比例。例如Startlifetime=5s，lifetime=0.1，则拖尾长度为粒子0.5s内走过的距离。

因为拖尾贴图和粒子本身贴图有重合，因此可以把拖尾的颜色开头部分设置为透明

继承粒子颜色和生命周期内颜色定义颜色A，拖尾颜色定义颜色B，拖尾最终呈现颜色为A*B

Texture Sheet Animation（纹理表格动画）子模块

将Renderer模块中的材质切分成多块，播放。

一般用于制作火焰效果。此时一般关闭发射器形状，设置预热和循环，速度为零，发射间隔根据粒子生存时间调整（例如粒子生存5s，则发射速率为0.2，此时粒子消失后下一个粒子刚好出现）

由上面图片制作的材质需要网格形式8*8切分，切出来的64帧会在一个粒子周期内播放 X 轮（默认1轮 Cycle参数）

Light（灯光）子模块

让发射的粒子携带一个光源

  • Light选择点光源
  • Ratio 比例，例如0.1表示有10%的粒子会携带一个光源 - Use Particle Color，使用粒子的颜色作为灯光颜色
  • Range / Intensity Mulitiplier，范围因子/强度因子
  • Max Lights，光源的最大数量

Sub Emission（二级发射器）子模块

二级粒子最好作为一级粒子的子节点，以便预览

其中， 什么时候发射二级粒子发射器? 当一级粒子死亡(Death )时,创建二级发射器 在一级粒子死亡的位置，创建二级粒子发射器 当Emit Probability = 1时，表示100%创建二级发生器。

二级粒子的发射时机:
Birth ，当一级粒子创建时
Death ，当一级粒子死亡时
Collision ，当一级粒子与其他物体发生碰撞时
其中大多数时候二级粒子发射方式必须使用爆发模式

Collison（碰撞）子模块

Collision模块，检测粒子与其他物体的碰撞。

粒子的碰撞检测，有两种类型: - Planes ，检测与目标平面之间的碰撞 - World，检测与世界中物体之间的碰撞( collider ) 其中，Planes方式的碰撞检测效率更高。

Planes,指定碰撞平面。所在位置添加一个碰撞平面，确定位置后，应将它隐藏
当指定为 World 时，目标物体应该添加Collider，用Layer来细化要碰撞的目标，尽量减少碰撞的范围

Dampen ，速度的衰减，o不衰减/1则100%衰减
Bounce，粒子在碰撞时弹跳的力度
Lifetime Loss ，生命时长的损失

粒子碰撞时，会调用一个事件函数OnParticleCollision
添加 SimpleLogic脚本，挂在粒子上
添加事件函数onParticleCollision
勾选 Send Collision Message
粒子事件无法预览
