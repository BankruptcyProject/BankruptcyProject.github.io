Posted on 2022-03-14 Edited on 2023-06-02

题目

You are given an array of k linked-lists lists, each linked-list is sorted in ascending order.

Merge all the linked-lists into one sorted linked-list and return it.

Constraints:

  • k == lists.length
  • 0 <= k <= 104
  • 0 <= lists[i].length <= 500
  • -104 <= lists[i][j] <= 104
  • lists[i] is sorted in ascending order.
  • The sum of lists[i].length will not exceed 104.

第一次提交(21%, 39%)

思路：新建一个链表存储答案，遍历所有输入链表的头，选取最小值，复制到新链表中。

是纯暴力。

分析时间复杂度，选取kn个结点，每次用时k，为O(k2n)；空间复杂度为O(kn)

合并两个有序列表(LC题解)

简化问题，思考合并两个有序列表

为了达到空间代价是 O(1)，宗旨是「原地调整链表元素的 next 指针完成合并」

可以理解为用一根绳子把两行结点重新串起来，不用新建一行复制品

原地顺序合并（10%，90%）

思路：仍然需要遍历所有链表头找到最小值，但不新建链表复制最小值，而是原地将现有结点串起来。

空间复杂度为O(1)，时间复杂度仍然为O(k2n)

分治，原地合并（50%，80%）

思路：二分法合并，将问题分解到每次合并两条链表。

思考：用空间换时间，对比顺序合并，相当于将k个表头的对比（一次时间为k的循环）分解存储在树中。

k条链表，每条长度为n。

第一次合并后剩下 $ {k } $ 条，合并一条用时2n，故第一次合并一共用时 $ {k } * 2n$ = kn

第二次合并后剩下 $ {k } $ ，合并一条用时4n，故第二次合并一共用时 $ {k } * 4n$ = kn

···

二分合并一共需要合并log2k次，每次合并用时kn，故总时间复杂度为O(kn·log2k)

二分递归，树的高度为log2k，故空间复杂度为O(log2k)
